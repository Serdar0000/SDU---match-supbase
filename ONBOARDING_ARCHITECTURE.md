## 🔄 Логика потока

```dart
// Модель данных онбординга
class OnboardingData {
  String? name;
  int? age;
  String? gender;        // 'male', 'female', 'other'
  String? lookingFor;    // 'male', 'female', 'all'
  String? faculty;
  int? yearOfStudy;      // 1-4
  String? photoUrl;
  
  bool get isComplete => 
    name != null && 
    age != null && 
    gender != null && 
    lookingFor != null &&
    faculty != null && 
    yearOfStudy != null && 
    photoUrl != null;
}

// Bloc Events
- OnboardingStarted
- NameUpdated(String name)
- AgeUpdated(int age)
- GenderSelected(String gender)
- LookingForSelected(String lookingFor)
- FacultySelected(String faculty)
- YearOfStudySelected(int year)
- PhotoUploaded(String url)
- OnboardingCompleted

// Bloc States
- OnboardingInitial
- OnboardingInProgress(int step, OnboardingData data)
- OnboardingPhotoUploading
- OnboardingComplete
- OnboardingError(String message)
```

## 💾 Сохранение

**Стратегия:** Сохраняем после каждого шага в состояние Bloc, финальное сохранение в Supabase на последнем шаге.

```dart
// После завершения всех шагов:
final profile = UserProfile(
  id: userId,
  email: userEmail,
  name: data.name!,
  age: data.age!,
  gender: data.gender!,
  lookingFor: data.lookingFor!,
  faculty: data.faculty!,
  yearOfStudy: data.yearOfStudy!,
  imageUrl: data.photoUrl!,
  interests: [], // Пустой массив, заполнится позже
  bio: '',       // Пустая строка, заполнится позже
);

await supabaseService.saveUserProfile(profile);
```
